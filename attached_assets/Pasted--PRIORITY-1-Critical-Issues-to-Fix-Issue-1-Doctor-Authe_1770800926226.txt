## PRIORITY 1: Critical Issues to Fix

### Issue 1: Doctor Authentication System
**Current Problem:** 
- Doctors enter their name and a code is generated each time
- Creates duplicate accounts/sessions
- Unclear authentication flow
- Patients only need a name (no real authentication)

**Required Solution:**
- Implement proper doctor registration with:
  - Email-based signup (send verification link to email)
  - Password creation (strong password validation)
  - One-time verification code via email (for security confirmation)
  - Doctor profile verification (optional: license/credential validation)
  
- Implement doctor login with:
  - Email + Password authentication
  - "Remember me" option
  - Password reset via email
  - Session persistence (user stays logged in until logout)

- Implement patient authentication with:
  - Patient signup: Full name + Email + Password
  - Patient login: Email + Password
  - Remove simple name-only login
  - Session persistence

- Backend Requirements:
  - Use proper authentication library (Firebase Auth, Auth0, or JWT with bcrypt hashing)
  - Store user sessions securely
  - Implement logout functionality
  - Add "forgot password" flow

---

### Issue 2: Keyboard Hiding Text Input Field
**Current Problem:**
- Virtual keyboard on mobile overlaps or hides the message input box

**Required Solution:**
- Implement keyboard-aware scroll behavior:
  - Detect keyboard show/hide events
  - Automatically scroll chat to bottom when keyboard appears
  - Adjust container height to prevent overlap
  - Use `android:windowSoftInputMode="adjustResize"` or equivalent for your framework
  
- If using React/Web:
  ```
  - Use Intersection Observer or scroll-to-bottom library
  - Add padding/margin when keyboard is active
  - Consider using `position: fixed` for input with proper z-indexing
  ```

- If using Flutter/Native:
  ```
  - Wrap inputs in Scaffold with resizeToAvoidBottomInset: true
  - Use FocusScope to manage focus
  - Implement ScrollController to auto-scroll on keyboard
  ```

---

### Issue 3: Chat Not Syncing Between Devices
**Current Problem:**
- Doctor shares code with patient
- Patient uses code to start chat
- Chat appears on patient's device but NOT on doctor's phone
- Messages not synchronized in real-time

**Required Solution:**
- Implement real-time database synchronization:
  - Use Firestore, Realtime Database, or Socket.io
  - When patient initiates chat with code, create chat session in database
  - Both doctor and patient subscribe to same chat room
  - Implement message listener that updates both devices instantly
  
- Chat room structure (database):
  ```
  /chats/{chatId}
    - chatCode: "ABC123" (unique code for this session)
    - doctorId: "doctor_uid"
    - patientId: "patient_uid"
    - createdAt: timestamp
    - lastMessage: timestamp
    - participants: {doctorId, patientId}
    
  /chats/{chatId}/messages/{messageId}
    - senderId: "user_uid"
    - text: "message content"
    - timestamp: ISO timestamp
    - read: boolean
  ```

- Implementation:
  - When doctor creates session: generate unique code, store in database with doctor's ID
  - When patient enters code: look up session, add patient to chat, subscribe to messages
  - Use real-time listeners on both ends to receive messages instantly
  - Test on multiple devices simultaneously

1.3 Session Management & Security
Session Persistence
After successful login:
- Store auth token securely (secure HTTP-only cookie or secure storage)
- User remains logged in until:
  ☐ They manually log out
  ☐ 30 days pass (if "Remember me" checked)
  ☐ They switch devices
  ☐ Session times out

Session timeout for medical data access:
- 15 minutes of inactivity → auto-logout
- Warning at 14 minutes: "Session expires in 1 minute"
- Button on warning: "Stay Logged In" (extends by 15 more minutes)
- Activity definition: scroll, click, type, tap (NOT just reading)
Login Attempt Protection
Rate Limiting:
- 5 failed login attempts → lock for 15 minutes
- Show message: "Too many failed attempts. Try again in 15 minutes"
- Alert user via email: "Multiple failed login attempts detected"

Security measures:
- Hash passwords with bcrypt (cost: 12 rounds)
- Never store plain text passwords
- Use strong session tokens (minimum 32 bytes)
- Implement CSRF protection on login forms
Logout
Logout Button Location:
- Settings page > Account > Log Out
- Click logout → confirm dialog

Dialog:
"Are you sure you want to log out?"
Buttons: "Log Out" | "Cancel"

After logout:
- Clear all session data
- Clear stored tokens
- Redirect to login page
- Show message: "You have been logged out successfully"
